# Data Model — Helmsman Database Schema

> Single source of truth for all database entities. Read this when touching any model.
> Package: `packages/db` — Prisma schema at `packages/db/prisma/schema.prisma`

---

## Entity Relationship Overview

```
Team (1) ──── (N) TeamMember
  │                   │
  │                   └── User (1) ──── (N) Conversation
  │                                            │
  │                                            ├── (N) Message
  │                                            └── (N) Plan
  │                                                      │
  │                                                      └── (N) PlanStep
  │
  ├── (N) Credential (encrypted)
  ├── (N) AuditEvent
  └── (N) ToolExecution
```

## Models

### User

Represents a person who interacts with Helmsman.

```prisma
model User {
  id            String       @id @default(cuid())
  telegramId    String?      @unique
  slackId       String?      @unique
  displayName   String
  email         String?      @unique
  role          UserRole     @default(OPERATOR)
  createdAt     DateTime     @default(now())
  updatedAt     DateTime     @updatedAt

  teamMembers   TeamMember[]
  conversations Conversation[]
  auditEvents   AuditEvent[]

  @@map("users")
}

enum UserRole {
  VIEWER        // read-only queries
  OPERATOR      // read + write with approval
  ADMIN         // full access + team management
}
```

### Team

A group of users sharing infrastructure credentials and settings.

```prisma
model Team {
  id            String       @id @default(cuid())
  name          String
  slug          String       @unique
  settings      Json         @default("{}")
  createdAt     DateTime     @default(now())
  updatedAt     DateTime     @updatedAt

  members       TeamMember[]
  credentials   Credential[]
  auditEvents   AuditEvent[]

  @@map("teams")
}

model TeamMember {
  id        String       @id @default(cuid())
  teamId    String
  userId    String
  role      TeamRole     @default(MEMBER)
  joinedAt  DateTime     @default(now())

  team      Team         @relation(fields: [teamId], references: [id], onDelete: Cascade)
  user      User         @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([teamId, userId])
  @@map("team_members")
}

enum TeamRole {
  MEMBER
  ADMIN
  OWNER
}
```

### Conversation

A chat session between a user and Helmsman. Maps to a Telegram chat.

```prisma
model Conversation {
  id              String    @id @default(cuid())
  userId          String
  platform        Platform  @default(TELEGRAM)
  platformChatId  String    // Telegram chat_id or Slack channel_id
  status          ConversationStatus @default(ACTIVE)
  metadata        Json      @default("{}")
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt

  user            User      @relation(fields: [userId], references: [id])
  messages        Message[]
  plans           Plan[]

  @@unique([platform, platformChatId])
  @@index([userId, updatedAt])
  @@map("conversations")
}

enum Platform {
  TELEGRAM
  SLACK
}

enum ConversationStatus {
  ACTIVE
  ARCHIVED
}
```

### Message

A single message in a conversation (from user or agent).

```prisma
model Message {
  id              String      @id @default(cuid())
  conversationId  String
  role            MessageRole
  content         String      // the text content
  correlationId   String      // traces request end-to-end
  platformMsgId   String?     // Telegram message_id for dedup
  metadata        Json        @default("{}") // intent, tokens used, etc.
  createdAt       DateTime    @default(now())

  conversation    Conversation @relation(fields: [conversationId], references: [id], onDelete: Cascade)

  @@index([conversationId, createdAt])
  @@index([correlationId])
  @@index([platformMsgId])
  @@map("messages")
}

enum MessageRole {
  USER
  ASSISTANT
  SYSTEM
  TOOL
}
```

### Plan

A plan generated by the agent for a user's action request.

```prisma
model Plan {
  id              String      @id @default(cuid())
  conversationId  String
  intent          String      // classified intent (QUERY, ACTION, DEBUG, etc.)
  summary         String      // human-readable plan summary
  riskTier        RiskTier
  status          PlanStatus  @default(PENDING)
  approvedAt      DateTime?
  rejectedAt      DateTime?
  completedAt     DateTime?
  failedAt        DateTime?
  failureReason   String?
  metadata        Json        @default("{}")
  createdAt       DateTime    @default(now())
  updatedAt       DateTime    @updatedAt

  conversation    Conversation @relation(fields: [conversationId], references: [id])
  steps           PlanStep[]

  @@index([conversationId, createdAt])
  @@index([status])
  @@map("plans")
}

enum RiskTier {
  READ_ONLY       // Tier 1: no approval needed
  LOW_RISK        // Tier 2: announce and proceed
  SIGNIFICANT     // Tier 3: require explicit approval
  DESTRUCTIVE     // Tier 4: require typed confirmation
}

enum PlanStatus {
  PENDING         // waiting for approval
  APPROVED        // user approved, ready to execute
  EXECUTING       // currently running
  COMPLETED       // all steps succeeded
  FAILED          // one or more steps failed
  REJECTED        // user rejected the plan
  CANCELLED       // user or system cancelled
}
```

### PlanStep

An individual step within a plan, mapping to a tool execution.

```prisma
model PlanStep {
  id          String        @id @default(cuid())
  planId      String
  order       Int           // execution order (1, 2, 3...)
  tool        String        // tool identifier (e.g., "aws.ec2.stopInstance")
  action      String        // specific action within the tool
  params      Json          // parameters for the tool call
  status      StepStatus    @default(PENDING)
  result      Json?         // tool execution result
  error       String?       // error message if failed
  startedAt   DateTime?
  completedAt DateTime?
  createdAt   DateTime      @default(now())

  plan        Plan          @relation(fields: [planId], references: [id], onDelete: Cascade)

  @@index([planId, order])
  @@map("plan_steps")
}

enum StepStatus {
  PENDING
  RUNNING
  COMPLETED
  FAILED
  SKIPPED
}
```

### Credential

Encrypted storage for team infrastructure credentials.

```prisma
model Credential {
  id              String      @id @default(cuid())
  teamId          String
  provider        Provider
  label           String      // "AWS Production", "AWS Staging", etc.
  encryptedData   String      // AES-256-GCM encrypted JSON blob
  iv              String      // initialization vector
  authTag         String      // GCM authentication tag
  lastRotatedAt   DateTime?
  expiresAt       DateTime?
  createdAt       DateTime    @default(now())
  updatedAt       DateTime    @updatedAt

  team            Team        @relation(fields: [teamId], references: [id], onDelete: Cascade)

  @@unique([teamId, provider, label])
  @@map("credentials")
}

enum Provider {
  AWS
  GCP           // future
  GITHUB        // future
  KUBERNETES    // future
}
```

### AuditEvent

Immutable log of every significant action.

```prisma
model AuditEvent {
  id              String      @id @default(cuid())
  teamId          String
  userId          String?     // null for system events
  correlationId   String
  eventType       String      // "tool.executed", "plan.approved", etc.
  severity        Severity    @default(INFO)
  action          String      // human-readable action description
  resource        String?     // target resource ARN/ID
  planSnapshot    Json?       // plan state at time of event
  result          Json?       // outcome data
  metadata        Json        @default("{}")
  createdAt       DateTime    @default(now())

  team            Team        @relation(fields: [teamId], references: [id])
  user            User?       @relation(fields: [userId], references: [id])

  @@index([teamId, createdAt])
  @@index([correlationId])
  @@index([eventType, createdAt])
  @@index([userId, createdAt])
  @@map("audit_events")
}

enum Severity {
  DEBUG
  INFO
  WARN
  ERROR
  CRITICAL
}
```

### ToolExecution

Tracks individual tool calls with timing and results.

```prisma
model ToolExecution {
  id              String      @id @default(cuid())
  correlationId   String
  tool            String      // "aws.ec2.describeInstances"
  action          String
  params          Json
  result          Json?
  errorCode       String?
  errorMessage    String?
  durationMs      Int?
  createdAt       DateTime    @default(now())

  @@index([correlationId])
  @@index([tool, createdAt])
  @@map("tool_executions")
}
```

## Indexing Strategy

| Table | Key Queries | Indexes |
|-------|------------|---------|
| `messages` | Load last N messages for conversation context | `(conversationId, createdAt)` |
| `messages` | Dedup by platform message ID | `(platformMsgId)` |
| `plans` | Find pending plans for approval | `(status)` |
| `audit_events` | Team audit log (paginated, time-sorted) | `(teamId, createdAt)` |
| `audit_events` | Trace a single request | `(correlationId)` |
| `tool_executions` | Trace tool calls for a request | `(correlationId)` |

## Migration Rules

1. Every schema change requires a named migration: `bunx prisma migrate dev --name <description>`
2. **Never edit** a migration file after it's been applied to any environment
3. **Never use** `prisma db push` in production (dev only for rapid iteration)
4. Destructive migrations (drop column, drop table) require a 2-step process:
   - Step 1: Make column nullable / stop writing to it
   - Step 2: Drop in next migration after confirming no reads
5. Always add `@@index` for columns used in `WHERE` or `ORDER BY` clauses
6. JSONB columns (`Json` type) should have a Zod schema documenting their structure in the codebase

## Seed Data

Seed script at `packages/db/prisma/seed.ts`:
```typescript
import { prisma } from "../src/client";

async function seed() {
  // Create a test team
  const team = await prisma.team.create({
    data: { name: "Demo Team", slug: "demo" },
  });

  // Create a test user
  const user = await prisma.user.create({
    data: {
      telegramId: "123456789",
      displayName: "Demo User",
      email: "demo@helmsman.chat",
      role: "ADMIN",
    },
  });

  // Link user to team
  await prisma.teamMember.create({
    data: { teamId: team.id, userId: user.id, role: "OWNER" },
  });
}

seed();
```
